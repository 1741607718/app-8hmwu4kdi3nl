# API 数据量分析报告

## 1. API 端点概览

### 车辆通行数据 API
- **端点**: `/api/vehicle`
- **功能**: 获取车辆通行记录（车流和车速信息）
- **参数**: 
  - `startDate`: 开始日期 (格式: YYYY-MM-DD)
  - `endDate`: 结束日期 (格式: YYYY-MM-DD)
- **返回字段**:
  - `cph`: 车牌号
  - `qcysdm`: 识别状态代码
  - `qcysmc`: 识别状态名称
  - `sbtdbm`: 站点代码
  - `sbtdmc`: 站点名称
  - `zpsj`: 通过时间
  - `cs`: 车速 (可选)
  - `cjsj`: 创建时间 (可选)
  - `cllx`: 车辆类型 (可选)

### 车辆登记数据 API
- **端点**: `/api/vehicle-registration`
- **功能**: 获取车辆登记信息
- **参数**:
  - `startDate`: 开始日期 (格式: YYYY-MM-DD)
  - `endDate`: 结束日期 (格式: YYYY-MM-DD)
- **返回字段**:
  - `guid`: GUID
  - `gh`: 工号 (可选)
  - `bh`: 编号 (可选)
  - `xm`: 姓名 (可选)
  - `lxfs`: 联系方式 (可选)
  - `bm`: 部门 (可选)
  - `cllx`: 车辆类型 (可选)
  - `cp`: 车牌
  - `djrq`: 登记日期 (可选)
  - `dqrq`: 到期日期 (可选)
  - `dxwb`: DXWB (可选)
  - `syjf`: 剩余积分 (可选)

> **注意**: 根据代码分析，车辆登记API在内部代理服务中错误地使用了消防API的配置（FIRE_SAFETY_API_CONFIG）来调用相同的API端点（Service_armiBZ8u9xVcvpQ2iVxz），这可能导致无法获取正确的车辆登记数据。正确的实现应该使用独立的车辆登记API端点配置。在VehiclesPage.tsx页面中，车辆登记数据通过`/api/vehicle-registration`端点获取。

### 消防设备数据 API
- **端点**: `/api/fire-safety`
- **功能**: 获取消防设备检测数据
- **参数**:
  - `startDate`: 开始日期 (格式: YYYY-MM-DD)
  - `endDate`: 结束日期 (格式: YYYY-MM-DD)
- **返回字段**:
  - `dqrq`: 检测日期
  - `bh`: 设备编号
  - `syjf`: 设备状态
  - `dxwb`: 位置代码
  - `bm`: 位置名称

## 2. 数据量分析

### 车辆通行数据
- **日常数据量**: 根据模拟数据生成函数，每天约20-70条记录
- **数据特征**: 
  - 每天随机生成20-70条通行记录
  - 使用5个车牌号进行随机分配
  - 4个站点(东门、西门、南门、北门)进行随机分配
  - 时间随机分布在24小时内
- **时间范围影响**: 数据量与时间范围成正比

### 车辆登记数据
- **日常数据量**: 根据模拟数据生成函数，每天每个位置生成3-7条记录
- **数据特征**:
  - 每天在5个位置(教学楼A栋、教学楼B栋、宿舍楼1号、宿舍楼2号、图书馆)生成车辆登记记录
  - 每个位置每天生成3-7条车辆登记记录
- **时间范围影响**: 数据量与时间范围成正比

### 消防设备数据
- **日常数据量**: 根据模拟数据生成函数，每天约15-35条记录
- **数据特征**:
  - 每天在5个位置(教学楼A栋、教学楼B栋、宿舍楼1号、宿舍楼2号、图书馆)生成设备记录
  - 每个位置每天生成3-7条设备记录
  - 90%的设备状态为"正常"，10%为"异常"
- **时间范围影响**: 数据量与时间范围成正比

## 4. 数据统计功能

系统提供了数据统计功能：
- **车辆统计**: 总数、按日期分布
- **消防设备统计**: 总数、正常/异常设备数量
- **人员统计**: 从 `personnel_stats` 表获取
- **安保统计**: 从 `security_stats` 表获取
- **宿管统计**: 从 `dormitory_stats` 表获取

## 6. 注意事项

1. **内网访问限制**: 由于 `api.wzbc.edu.cn` 是内网地址，需要通过内部代理服务访问
2. **API响应格式**: 内网API返回格式为 `{ data: { Rows: [...] } }`，代理服务会提取实际数据数组
3. **错误处理**: 服务包含超时(10秒)和错误处理机制
4. **CORS支持**: 代理服务支持跨域请求
5. **车辆登记API配置问题**: 当前内部代理服务中，车辆登记API错误地使用了消防API配置，需要修正以使用正确的车辆登记API端点
